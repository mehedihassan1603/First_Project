Blog-1: Discuss the scope of var, let, and const

  • var is a Global scoped or function scoped.
  • It is Global Scoped because any variable that is declared with var outside a function block is available for use in the whole window.
  • It is a function scoped because function scoped when it is declared within a function, it is available and can be accessed only within that function.
  • var is a variable that can be re-declared and updated.
  • let is a block scoped.
  • It can be updated but not re-declared
  • It can't be accessible outside the particular block ({block})
  • const is also a block scoped.
  • It can't be updated or re-declared
  • When users declare a const variable, they need to initialize it, otherwise, it returns an error.

Blog-2: Tell us the use cases of null and undefined

  • Null in JavaScript means an empty value and is also a primitive type in JavaScript.
  • The variable which has been assigned as null contains no value.
  • Also, null is an object in JavaScript. When it gets assigned to a variable, it represents no value.
  • Undefined, means the variable has been declared, but its value has not been assigned.
  • When we declare a variable but do not assign a value to it, the variable becomes undefined.
  • When we do not pass an argument for a function parameter, the default value is taken as undefined.
  • Besides, when a function doesn't return a value, it returns undefined.

Blog-3: What do you mean by REST API?

A REST API, which stands for Representational State Transfer Application Programming Interface, is a set of rules and conventions for building and interacting with web services.
It's a way for different software applications to communicate with each other over the internet.
For example, if I use a mobile app to fetch data from a server, that app is making requests to a REST API.
The app sends a GET request to a specific URL (resource), and the server responds with data in a specified format (representation), often in JSON. This allows the app to retrieve and display information from the server, update data, or perform other actions based on the API's rules.

So, a REST API is a standardized way for software applications to talk to each other over the internet, making it easier to access and exchange data between different systems
